Technical Architecture

Component structure, state management, and performance approach

Component Structure

UnifiedAIAssistant.jsx
├── state: { isOpen, isExpanded, mode, messages }
├── Compact Widget
├── 70×70px circle button
├── Hover animations (scale, shadow)
└── Click → expand animation
├── Expanded Panel (Glassmorphism)
├── Smooth grow animation (70px → 380px width)
├── Header with mode toggle (Text ↔ Voice)
├── Content Area (switches based on mode)
├── TextChat (messages + input)
└── VoicePanel (waveform + visualizer)
└── Controls & Close button
└── useVoiceSession hook (reuse existing)

State Management

Component State

  • isOpen - Widget visible
  • isExpanded - Panel expanded
  • mode - 'text' | 'voice'
  • messages - Chat history
  • isListening - Voice recording

Transitions

  • Compact ↔ Expanded (0.3s)
  • Text ↔ Voice (0.2s)
  • Message fade-in (0.15s)
  • Waveform animate (continuous)

Performance Optimizations

  • Lazy load voice module: Only load audio/waveform logic when voice mode is activated
  • Memoized components: TextChat and VoicePanel use React.memo to prevent unnecessary re-renders
  • Minimal backdrop blur: Use CSS backdrop-filter (hardware accelerated) instead of canvas blur
  • Debounced input: Message input debounced 300ms to reduce API calls
  • Virtual scrolling: If chat grows large, only render visible messages
  • CSS animations over JS: Use CSS @keyframes for waveform pulse (60fps, GPU-accelerated)

Key Features

Visual Polish

  • Glassmorphism (backdrop blur + opacity)
  • Gradient accents (purple/indigo)
  • Smooth transitions (all 0.2-0.3s)
  • Animated waveforms (real-time)
  • Hover states on all buttons

Interaction Model

  • Click bubble → expand
  • Click X → collapse (not close)
  • Click mode buttons → smooth transition
  • Voice visualizer animates while listening
  • Messages fade in as they arrive

Integration Points

  • Reuse existing: AITalkPanel's API calls (assistantPipelineService), mentoringService
  • Reuse existing: useVoiceSession hook from AIVoicePanel
  • Replace: Remove old AITalkPanel and AIVoicePanel after this is deployed
  • Layout.jsx: Drop UnifiedAIAssistant component at app root (floats above everything)
  • No major refactoring needed: This is additive; old components can coexist during transition

Browser Support & Responsiveness

  • Desktop: 380px panel fixed to bottom-right, floats over content
  • Tablet: 340px panel, positioned bottom-right with safe margins
  • Mobile: Panel takes 90vw width, positioned at bottom (above keyboard), can slide up if needed
  • Backdrop blur: Gracefully degrades on older browsers (just uses opacity)
  • Animations: prefers-reduced-motion respected (disable animations if enabled)
